## Story
> Como usuário,
> Quero configurar mensagens automáticas para cada stage do Kanban com período definido,
> Para que leads recebam follow-up sem intervenção manual.

## Dev Notes

### API Contract — Create Auto Message
```
POST /api/auto-messages
Body: {
  "stage_id": "uuid",
  "message_template": "Olá {{name}}, tudo bem? Vi que...",
  "message_type": "text",
  "media_url": null,
  "delay_minutes": 60,
  "is_active": true,
  "position": 0
}
```

### Delay Conversion (Frontend → Backend)
| Input | Unit | delay_minutes |
|-------|------|---------------|
| 1 | hora | 60 |
| 24 | horas | 1440 |
| 3 | dias | 4320 |
| 30 | minutos | 30 |

Frontend converte para `delay_minutes` antes de enviar ao backend.

### Preview
O preview renderiza o template substituindo variáveis por dados de exemplo:
- `{{name}}` → "João Silva"
- `{{phone}}` → "11999999999"
- `{{company}}` → "Empresa X"
- `{{stage}}` → "Qualificado"

### Testing
- Criar auto message para stage "Qualificado" com delay 60min → salva no Supabase
- Criar 3 mensagens (1h, 24h, 72h) → listadas em ordem
- Desativar mensagem → toggle para is_active = false
- Preview exibe mensagem formatada com dados de exemplo
- Editar delay e template → dados atualizados
