## Dev Notes

### Webhook Payload — messages.upsert
```json
{
  "event": "messages.upsert",
  "instance": "leadtrack-main",
  "data": {
    "key": {
      "remoteJid": "5511999999999@s.whatsapp.net",
      "fromMe": false,
      "id": "3EB0A0B0C0D0E0F0"
    },
    "message": {
      "conversation": "Olá, gostaria de mais informações",
      "extendedTextMessage": {
        "text": "Mensagem com link preview"
      },
      "imageMessage": {
        "url": "https://...",
        "caption": "foto do produto",
        "mimetype": "image/jpeg"
      }
    },
    "messageTimestamp": "1679000000"
  }
}
```

### Phone Extraction
```javascript
const phone = message.key.remoteJid.replace('@s.whatsapp.net', '');
// "5511999999999@s.whatsapp.net" → "5511999999999"
```

### Message Content Extraction Priority
```javascript
const content = message.message?.conversation
  || message.message?.extendedTextMessage?.text
  || message.message?.imageMessage?.caption
  || '';
```

### Webhook Set Body
```json
{
  "enabled": true,
  "url": "https://your-server.com/api/webhook/messages",
  "webhookByEvents": true,
  "webhookBase64": false,
  "events": ["MESSAGES_UPSERT", "CONNECTION_UPDATE"]
}
```

### Environment Variable
```
WEBHOOK_URL=https://your-public-url.com/api/webhook/messages
```
Em desenvolvimento, usar ngrok: `ngrok http 3001`

### Realtime Subscription
```javascript
supabase
  .channel('new-messages')
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'message_log',
    filter: 'direction=eq.incoming'
  }, handler)
  .subscribe();
```

### Testing

- Mensagem com imagem → salva com media_url

- 2 tabs abertas → mensagem chega em ambas via Realtime
