## Dev Notes

### API Contract — List Leads
```
GET /api/leads?page=1&limit=20&search=João&stage=uuid&tags=hot,warm
Response: {
  data: [leads],
  pagination: { page, limit, total, totalPages }
}
```

### API Contract — Create Lead
```
POST /api/leads
Body: { name, phone, email?, company?, tags?, notes? }
Response: { lead } (with current_stage_id = first stage)
```

### Auto-assign First Stage
Na criação, buscar stage com menor `position` e `is_active = true`. Inserir lead com `current_stage_id` e criar entry em `lead_stage_history` (from_stage_id: null, to_stage_id: first_stage).

### Search Query
```sql
WHERE (name ILIKE '%search%' OR phone ILIKE '%search%' OR email ILIKE '%search%')
AND (stage filter)
AND (tags filter — use array overlap: tags && ARRAY['tag1','tag2'])
ORDER BY created_at DESC
LIMIT 20 OFFSET (page-1)*20
```

### Testing
- Criar lead com telefone → aparece no primeiro stage do Kanban
- Buscar por nome parcial → resultado filtrado
- Filtrar por stage → mostra apenas leads daquele stage
- Editar lead → dados atualizados
- Excluir lead → removido do Kanban e da lista
- Criar lead sem telefone → erro de validação
