## Acceptance Criteria

## Dev Notes

### Scheduler Flow
```
Every 1 minute:
1. SELECT scheduled_messages WHERE status = 'pending' AND scheduled_at <= NOW() LIMIT 50
2. JOIN leads (check auto_followup_paused)
3. JOIN auto_messages (get template, type, media_url)
4. For each:
   a. If lead.auto_followup_paused → skip
   b. Send via Evolution API
   c. If success → status = 'sent', INSERT message_log
   d. If failure → attempts++, if attempts >= 3 → status = 'failed'
      else → scheduled_at = NOW() + 2^attempts minutes
```

### Schedule Creation (on stage change)
```javascript
// Cancel old stage messages
await supabase
  .from('scheduled_messages')
  .update({ status: 'cancelled' })
  .match({ lead_id: leadId, stage_id: oldStageId, status: 'pending' });

// Fetch auto messages for new stage
const { data: autoMessages } = await supabase
  .from('auto_messages')
  .select('*')
  .eq('stage_id', newStageId)
  .eq('is_active', true);

// Create scheduled messages
const scheduled = autoMessages.map(am => ({
  lead_id: leadId,
  auto_message_id: am.id,
  stage_id: newStageId,
  scheduled_at: new Date(Date.now() + am.delay_minutes * 60000).toISOString(),
  status: 'pending'
}));

await supabase.from('scheduled_messages').insert(scheduled);
```

### Retry Backoff
| Attempt | Wait | Total from original |
|---------|------|-------------------|
| 1 | 2 min | +2 min |
| 2 | 4 min | +6 min |
| 3 | FAIL | — |
