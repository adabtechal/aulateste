## Dev Notes

### Evolution API Endpoints
| Operation | Method | Endpoint | Auth |
|-----------|--------|----------|------|
| Create | POST | `/instance/create` | `apikey` header |
| QR Code | GET | `/instance/connect/{instance}` | `apikey` header |
| Status | GET | `/instance/connectionState/{instance}` | `apikey` header |
| Delete | DELETE | `/instance/delete/{instance}` | `apikey` header |
| Webhook | POST | `/webhook/set/{instance}` | `apikey` header |

### Create Instance Body
```json
{
  "instanceName": "leadtrack-main",
  "integration": "WHATSAPP-BAILEYS",
  "qrcode": true,
  "rejectCall": false,
  "groupsIgnore": true,
  "alwaysOnline": false,
  "readMessages": false,
  "readStatus": false,
  "syncFullHistory": false
}
```

### QR Code Response
```json
{
  "pairingCode": "string",
  "code": "string",   // ← this is the QR code data
  "count": 0
}
```
Renderizar `code` com `qrcode.react` library.

### Connection State Response
```json
{
  "instance": {
    "instanceName": "leadtrack-main",
    "state": "open"   // "open" = connected, "close" = disconnected
  }
}
```

### Environment Variables
```
EVOLUTION_API_URL=https://your-evolution-instance.com
EVOLUTION_API_KEY=your_api_key
```

### Polling Strategy
- QR Code: refetch every 30s while `state !== 'open'`
- Connection Status: refetch every 10s (useQuery refetchInterval)

### Testing
- Criar instância → dados salvos no Supabase
- QR code exibido → escanear com WhatsApp → status muda para "open"
- Deletar instância → removida do Supabase e Evolution API
- Instância desconectada → botão reconectar → novo QR code
